Runtime design

The hot path stays small.

BRG is designed like production infrastructure. The runtime path should only do what is necessary to execute a request. Everything else happens asynchronously.

Hot Path
  • Authenticate
  • Normalize
  • Cache lookup
  • Route
  • Execute
  • Normalize response
  • Return

Synchronous. Latency-sensitive. Sub-millisecond overhead.

Off Path
  • Telemetry
  • Token accounting
  • Cost engine
  • Savings calculator
  • Dashboard updates
  • Historical reporting

Asynchronous. Can degrade gracefully without blocking inference.

Visibility should never block inference.

Failure philosophy

Inference comes first.

BRG should fail predictably. If supporting systems degrade, inference should continue whenever possible.

If telemetry fails

Continue serving traffic

Buffer or drop non-critical telemetry according to policy.

If accounting lags

Continue serving traffic

Reconcile economics off-path.

If cache fails

Bypass cache

Call the provider directly.

If provider fails

Apply configured fallback

Apply configured fallback or fail-open behavior.

Visibility degradation is preferable to inference disruption.
