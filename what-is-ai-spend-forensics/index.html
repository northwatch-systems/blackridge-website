Category definition

What is AI token forensics?

Provider bills show totals. Token forensics shows causes.

It helps teams find where AI spend is leaking across workflows, retries, fallbacks, duplicate requests, model choices, apps, teams, tenants, and missing attribution.

How it differs from a basic AI gateway

Most AI gateways route, secure, and meter model traffic. Token forensics infrastructure answers the next question: which runtime behavior caused the bill, and what evidence supports the answer?

How it differs from provider dashboards

Provider dashboards show usage and invoice totals. Token forensics connects that cost back to workflows, retries, fallbacks, duplicate requests, model choices, apps, teams, tenants, and confidence coverage.

How it differs from observability and tracing

Observability and tracing tools are useful for debugging, latency, errors, and quality. Token forensics focuses specifically on spend causality, waste classes, attribution, evidence-backed safe tests, and ongoing visibility.

Common waste classes

Retry amplification

One action fans out into repeated model calls.

Fallback tax

Failure paths route to more expensive models or providers.

Duplicate inference

Similar work is paid for repeatedly.

Unattributed spend

Costs cannot be assigned to a team, app, tenant, or workflow.

Premium-model overuse

Expensive models become defaults for work that may not require them.

Abandoned branches

Agent branches spend tokens without affecting the final answer.

How Blackridge fits

Blackridge is a near real-time Token Economics and Forensics Engine for LLM and agent workflows. The one-workflow assessment uses logs, traces, gateway exports, provider events, runtime metadata, and scoped runtime evidence to produce Blackridge findings that platform, finance, executives, and application teams can inspect.

The purpose is proof before platform commitment: find where spend is leaking, inspect the rows behind the findings, and decide whether ongoing Blackridge forensics is justified. Blackridge explains and recommends; it does not take action or enforce changes for the user today.
