Category definition

What is AI spend forensics?

AI spend forensics is the practice of reconstructing token economics across LLM and agent workflows to explain where AI spend went, why it happened, what was waste, and what engineering should fix first.

How it differs from an AI gateway

AI gateways route, secure, and manage model traffic. AI spend forensics answers a different question: why did this workflow cost so much, who or what owns the spend, and which fix should move first?

How it differs from provider dashboards

Provider dashboards show usage and invoice totals. Spend forensics connects that cost back to workflows, retries, fallbacks, duplicate requests, model choices, apps, teams, tenants, and confidence coverage.

How it differs from observability and tracing

Observability and tracing tools are useful for debugging, latency, errors, and quality. Spend forensics focuses specifically on token economics, waste classes, attribution, and evidence-backed remediation.

Common waste classes

Retry amplification

One action fans out into repeated model calls.

Fallback tax

Failure paths route to more expensive models or providers.

Duplicate inference

Similar work is paid for repeatedly.

Unattributed spend

Costs cannot be assigned to a team, app, tenant, or workflow.

Premium-model overuse

Expensive models become defaults for work that may not require them.

Abandoned branches

Agent branches spend tokens without affecting the final answer.

How Blackridge fits

Blackridge is a token economics forensics system for LLM and agent workflows. It ingests logs, traces, gateway exports, provider events, runtime metadata, and optional runtime collection data to produce spend maps, waste findings, attribution coverage, confidence notes, and ranked remediation evidence.
