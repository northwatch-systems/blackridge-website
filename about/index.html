Overview

A software company focused on production AI economics.

Northwatch Systems works on the operational gap between AI applications and the financial systems that pay for them. Companies are adding language-model calls to support tools, copilots, document workflows, internal agents, and customer-facing products. The resulting spend is often visible before the cause is understood.

Blackridge is being built to make that cause easier to inspect. The product is intended for engineering, platform, finance, and application teams that need a shared view of AI usage without exposing sensitive prompts or relying only on monthly provider invoices.

What we build

Blackridge organizes AI usage records into cost evidence.

Blackridge is not a replacement for observability, finance tooling, or model-provider billing. It sits between those views and produces cause files: structured explanations of where AI cost came from, what evidence supports the finding, and what remains uncertain.

The work starts with specific questions: which workflow caused a spend increase, whether repeated calls were useful, where attribution is missing, which model choices changed cost, and whether a suspected waste pattern can be proven from available data.

Operating principles

How the product is designed to behave.

Evidence before recommendation

The system should show what it observed before suggesting what to change.

Uncertainty stays visible

Missing attribution, missing prices, sampled logs, and modeled estimates should be labeled clearly.

Sensitive data is treated as unnecessary by default

Cost accountability should not require broadcasting prompts, documents, or customer content across an organization.

Engineering and finance need the same object

A useful finding should be detailed enough for engineers and legible enough for budget owners.

Current focus

Blackridge is currently focused on deployed AI workflows.

The most relevant cases are workflows already running in production: support automation, internal copilots, document-processing pipelines, agent workflows, and customer-facing AI features with unclear cost behavior.

Initial work is narrow. We look at the evidence already available for one workflow, determine whether a useful cause file can be produced, and define the boundaries for any deeper technical review.
